
This computes whether the "Diff" tab on the Helm chart install page is disabled: if the user has made no edits there is nothing to compare, so it greys the tab out by string-comparing the chart's baseline values YAML (`originalYamlValues`) against the user's current values - either the form object re-serialized with `jsyaml.dump(this.chartValues)`, or the raw edited `valuesYaml`. The comment flags that a raw `===` between two YAML strings is inherently fragile (key order, quoting, indentation, or trailing newlines can differ for semantically identical docs), so the author is openly surprised it holds. It works only because `originalYamlValues` is itself seeded from the same serializer (line 378: `this.originalYamlValues = this.valuesYaml`), so an unedited form round-trips back to a byte-identical string; the documented fix for when that breaks is to normalize both sides through a "heavier stringify/jsyaml.dump" before comparing. It was introduced by Richard Cox in commit `2982aea0faa40c17292837dacf38699e9fb6f5ef` ("Change helm values form/values button group", 2021-05-14) as part of PR #2876 ("Add Helm Chart Install Stepper"), originally written as `jsyaml.safeDump` and later mechanically switched to `jsyaml.dump` when js-yaml was upgraded to v4 (which folded `safeDump` into `dump`). The constraint still holds five years on: the line is unchanged in current HEAD `efeb5d3` (2026-07-07), having survived two file moves (`pages/…/apps/install.vue` -> `pages/…/apps/charts/install.vue` -> `shell/…`) and the js-yaml upgrade, and nobody has ever had to apply the heavier-stringify remediation.
