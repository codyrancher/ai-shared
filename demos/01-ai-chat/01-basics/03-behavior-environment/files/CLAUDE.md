# Writing style
- Never use em dashes in any output: code, comments, commit messages, PR descriptions, or chat replies. Use a regular dash, parentheses, or split the sentence instead.
- When writing a commit message stick to this structure:
    - A 1 line summary at the top
    - 1 empty line
    - Fixes $ISSUE_NUMBER
        - Only if $ISSUE_NUMBER is available
    - A bulleted list that concisely describes the currently staged changes.
    - Highlight any unconventional choices in the change and concisely explain why we made them.
